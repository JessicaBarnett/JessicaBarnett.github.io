<h2>Available Icons</h2>

<p>these icons are all added as svg markup in a react component.  Therefore they are styleable via `fill`, border, and other svg properties.  </p>

<p>SVG elements have 2 styleable classes applied: `icon` and  `icon-*`, where `*` is the kebab-case svg file name (ie, `super-cool-cat.svg` yields `class="icon super-cool-cat-icon"`</p>

<h2>Adding Icons</h2>

<p>In order to add a new svg icon, do the following: </p>

<ol>
  <li>Add a `.svg` file to the `public/assets/icons` directory.</li>
  <li>run the `genIcons` npm script.</li>
  <li>verify the new Icon component and story in `src/icons`</li>
  <li>Verify new Icon appears here in the docs</li>
</ol>

<p>The genIcons script is a node script is located in `scripts/`.  It will grab all `.svg` files in `public/assets/icons`, and then pop that html into a component as JSX in `src/icons`.  It will also create a story file for the new component, and create a manifest to use in this documentation.</p>

<h2>SVG Icon File Guidelines</h2>
  <ul>
    <li>filename should be kebab-case. The script will convert it to CamelCase when naming the component. (ex: `super-cool-cat.svg` will result in component `SuperCoolCatIcon.tsx`)</li>
    <li>use `viewBox="0 0 280 280"` to keep the new icon in line with the others</li>
    <li>The xml-to-jsx conversion is very rudimentary.  Check the `genIconComponents` `xmlToJsx` function to add additional replacements.</li>
    <li>remove any `fill` attributes if you want the icon to be recolorable!</li>
  </ul>
